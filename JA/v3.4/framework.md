# Framework

## Project Sprintとはどのようなものか

Project Sprintは、未来を予測しにくい環境において、プロジェクトチームが小さく確実に成果を積み重ねていくためのフレームワークである。このフレームワークを活用することで、プロジェクトチームはプロジェクトを取り巻く環境についての最新の情報を手に入れ、その情報をもとに状況判断と軌道修正を重ねながら成果を生み出しつづけることができる。Project Sprintは、このフレームワークをプロジェクトの現場で実践するための方法やノウハウを併せて提供する。

Project Sprintは、さまざまなプロジェクトに取り組む中で得られた経験や知恵を集め抽象化して、抽出された共通点をもとに形作られている。そのためProject Sprintは、個々のプロジェクトに取り組む際の具体的な手順を指示するものにはなっていない。その代わりにProject Sprintは、複雑で個別性の高いプロジェクトに汎用的に適用できるような、ものごとの捉え方・考え方や、最適化を促進するための仕組みを提供する。Project Sprintは、それぞれのプロジェクトチームが自身の強みや既存のツールを活用しながら、よりいっそう自律性や創造性を発揮して目指す成果を生み出せるよう支援するものである。

## Project Sprintにおいて「プロジェクト」とはどのようなものか

Project Sprintにおける**プロジェクト**は、いわば「小さな実験の繰り返し」である。ここでいう実験とは、少し先の未来における小さくシンプルな成果の実現に向けて、確実に手を動かして何かを生み出してみるということを指す。不確実な状況であっても、少し先の未来のことは遠い未来のことよりも予測しやすい。未来の予測が難しいからこそ、実験的に小さな一歩を踏み出してみる必要がある。

この実験によって小さな成果が実現されれば、それはもちろんプロジェクトの達成に向けた一歩となる。しかしそれだけではなく、実験に取り組むことでプロジェクトや周囲の環境に関する情報（**環境情報**）が少しずつ明らかになり、次に取り組むべき課題や新たな選択肢が見えてくることがある。つまり、少し先の未来の不確実性が少し和らぐのだ。

もちろん、少し先の未来に達成されているべき成果を設定するためには、プロジェクトの目的や背景を踏まえて最終的に求める成果（**最終成果**）を定めておく必要がある。その最終成果を踏まえて、中期的な目標である中間成果や、日々の実験に落とし込みやすいシンプルな初期成果が設定される。

もしも未来が予測可能なものであれば、最終的に求める成果から単純に逆算して、確実なロードマップを引くことができるだろう。しかし、実際にはどのルートが最適かは不確かで、一度最適だろうと判断したルートも、環境の変化によって歩みにくくなったり望まぬ結果に導かれるものになったりするかもしれない。実験は、この環境の変化に気づくチャンスでもある。

プロジェクトでは、こうして小さな実験を重ね小さく成果を生み出すことで、環境に適応し未来を予測しながら、最終的な成果の達成へと一歩ずつ向かっていく。

## Project Sprintにおいて「活動」とはどのようなものか

Project Sprintでは、「ある状態を目指して行動する」ことを**活動**と呼んでいる。プロジェクトは、最終的な成果の実現を目指して実験を続けるという意味で、Project Sprintにおける活動の最も大きな単位である。その他にも、次のようなものはすべて活動と言える。

- プロジェクト内でいくつかのグループに分かれてそれぞれの目標達成を目指す
- 解決したい課題をミーティングのアジェンダとして提出してチームで話し合う
- 自律的なチームとなるべく役割を分担して個々の責任を果たす

すべての活動は、出力と成果から成る。
- **出力**: プロジェクトチームが自律的・自己完結的に取れる行動
- **成果**: プロジェクトチームが外部のステークホルダーに提供できる価値

出力と成果の関係は次のようなものである。

1. プロジェクトチームは、プロジェクトの目的や背景を踏まえて最終成果を定め、それをさらに中間成果及び初期成果に分解する。

2. プロジェクトチームは、1で定めた成果の実現に貢献しうる出力目標を設定する。
  - 出力目標は、必ず達成すべきものとしてコミットされなくてはならない
  - 出力目標の期日は、長くとも2か月以内を目安とする
  - 出力目標は達成したい成果を踏まえて設定されるが、必ずしも特定の成果に紐づいている必要はない

3. プロジェクトチームは、出力目標を達成するための行動を自律的に設定し遂行する。

4. プロジェクトチームによる出力もまた小さな実験であり、次の2つのインパクトを持ちうる。
  - 出力に取り組むことで得られた成果が、最終成果の実現に貢献する
  - 出力に取り組むことで得られた環境情報が、一度設定された成果を最適化する材料になる

5. プロジェクトチームは、上のインパクトに従って成果の予測の確度を高め出力目標やタスクの設定を適性化する。

この関係に則ってプロジェクトにおける活動に取り組むことにより、プロジェクトチームは少し先の未来の不確実性を和らげ、直近の行動を最適化することができる。ひいては、変化しつづける環境と目的に常に適応し、最適な成果を生み出す効果の高い出力に取り組むことが可能になる。

ただし、出力目標の達成は、ただちに特定の成果に結実しない場合もある。出力目標の設定時点では環境情報の取得が不十分であり、出力に取り組んでみてはじめて、その結果がプロジェクト外のステークホルダーに対する価値に繋がらないと分かることもあるからである。

Project Sprintでは、プロジェクトは特定のリーダーの指揮のもとで進むものではなく、自律性と主体性を持ってメンバー全員で進めるものだと考えている。それは単に作業を分担するということにとどまらない。プロジェクトの成果、つまりプロジェクトが外部に提供したい価値への認識をチームで共有し、自分たちが取り組む出力とその先にあるべき成果を全員が結びつけて捉えられるということを指す。

## プロジェクトの要素

Project Sprintでは、プロジェクトを以下の3つの要素からなると捉える。

### プログレス
プログレスは、プロジェクトにおける成果の実現を目的とし、そのための成果の設定と出力への取り組みにフォーカスする。

プロジェクトチームは、プロジェクトチーム自身の意思に基づいて一連の成果を設定し、それに対応する出力に取り組む。プロジェクトにおける成果の実現は、イテレーティブかつインクリメンタルなものでなくてはならない。つまり、一定の期間で区切って進捗や全体像、取るべき道筋を見直しながら、小さな成果を段階的に生み出しつづけるということだ。成果の設定と出力への取り組みは、外部環境の変化やプロジェクト内で得られる洞察に応じて繰り返しアップデートされる。

### チーミング
チーミングは、自律的なプロジェクトチームの形成を目的とし、プロジェクトにおけるメンバー相互の関係にフォーカスする。

自律的なチームとは、ある事柄に対して自分がチームのために何をすべきかを各メンバーが判断でき、かつ実際に行動できる状態のことを指す。そのためには、各メンバーが自身の責任・役割を理解して納得し、相互に期待値を共有している必要がある。その根底にあるのはメンバー間の信頼であり、この信頼の構築に最も効果的なのは、出力に取り組むことである。実際に手を動かしてみることは、未来の不確実性を和らげる手段であるのと同様に、役割を引き受けてチームへの貢献を示すことによって信頼関係を醸成する手段でもある。

### プロセス
プロセスは、プログレスとチーミングを最適解してプロジェクトを円滑に推進することを目的とし、そのための仕組みや手続きにフォーカスする。

チームの各メンバーは、継続的に出力に取り組む中で、他のメンバーに伝えたいアイデアや違和感（**テンション**）を発見する。この発見を定期的・反復的なミーティングの場に持ち寄ることで、環境に対する認識をチームで揃え、課題に対してはチームで意思決定を行うことができる。その結果、成果の設定や出力への取り組みがアップデートされる（プログレスの最適化）とともに、各メンバーが自律的に次の行動を取ることができるようになる（チーミングの最適化）。

## 「プロジェクトの推進」とはどういうことか

Project Sprintにおいてプロジェクトの推進とは、ここまで述べてきたような実験と最適化を繰り返し行って、プロジェクトの最終成果の達成に向けて小さく確実に成果を積み重ねていくことを指す。

そのためには、プロセスがうまく機能してプログレスとチーミングが最適化されている必要がある。この3つの要素すべてに共通し最適化の材料になるのは、出力への取り組みである。成果を生み出すにも、チームの信頼関係を築くにも、改善のための気付きを得るにも、出力への継続的かつ自律的な取り組みが最も重要である。

各メンバーが自律的に出力に取り組むためには、次のようなことが必要である。

- 実現すべき成果がプロジェクトチーム全員によって合意・納得され、環境の変化に応じたアップデートも都度共通認識化されていること
- 各メンバーが自身の責任や役割を自覚して納得の上で引き受け、相互の期待値を共有していること

この2つを実現するにあたって、Project Sprintでは**プロジェクト定例会議**をプロジェクト推進の軸に据え、行動の起点とする。プロジェクト定例会議とは、定期的・反復的にチームメンバー全員で行われるミーティングである。

重要なのは、チームメンバー全員が一時的に同一の環境に固定されてリアルタイムで会話をすることにより、素早く効率的な認識合わせと全員にとって納得感のある意思決定という、同期的な会議でしか生まれない効果を生むことだ。これを定期的・反復的に実施して各メンバーの出力への取り組みを共有することにより、次のようなことが可能になる。

- 1. 環境情報や相互の役割への認識が揃い、共通の成果を目指すためのプロジェクトチームとしてのナラティブが生まれる
- 2. アイデアの共同創造や問題の共同解決により、プロジェクトチームとして次に目指す成果を定めるための小さな意思決定が下せる
- 3. 各メンバーが、実現すべき成果に納得し、自律的にその成果のための次の行動に向かうことができる

ミーティングの役割は、よりよい形で成果を生み出すために個人では判断しきれない事柄に対しチームとしての意思決定を行い、個々のメンバーの活動の前提を与えることであるとも言える。

また、プロジェクト定例会議はプロジェクトに一定のリズムを生み出す。このリズムがあることで、タスクの粒度が揃えやすくなったり作業計画が立てやすくなったりするため、各メンバーの行動の習慣化も促される。習慣化された行動は定期的・反復的な出力を生み、その出力がプロジェクトをアップデートしつづける材料となる。
